**反饋功能需求**

1. 使用者可以對 AI 回答按「好」或「不好」。
2. 若使用者按「不好」，可以填寫原因，例如：
   * 回答太籠統
   * 沒有引用資料
   * 答非所問
   * 格式不清楚
   * 內容不夠完整
   * 自訂文字回饋
   * 或是點選其他，出現輸入框輸入

3. 系統要記錄每次反饋資料，包含：
   * 使用者 ID
   * 原始問題
   * AI 回答
   * 使用者評價，好或不好
   * 使用者補充意見

4. 每個使用者要有自己的專家指令JSON設定檔。
   * 一開始給預設指令

5. JSON 設定檔要保存該使用者偏好的回答方式，例如：
   * 回答長度
   * 回答格式
   * 是否要列點
   * 是否要附參考資料
   * 偏好的專業角度
   * 常見錯誤修正方向

6. 系統要定期整理使用者的負面反饋。
7. LLM 根據反饋分析問題，產生新的 prompt 調整建議。
8. LLM 不可以直接修改整個系統 prompt，只能提出允許範圍內的 JSON patch。
9. 系統要檢查 LLM 產生的修改內容是否合法，避免使用者亂輸入造成 prompt injection。
10. 合法的修改先存成 draft，不要直接覆蓋正式設定。
11. 可以LLM確認 draft，是否採用回饋意見。
12. 之後該使用者再次提問時，系統直接讀取他的專家指令JSON，合併進最終prompt。
13. 系統要保留版本紀錄，方便回復舊版設定。
14. 要防止亂用，例如：
* 禁止修改全域規則
* 禁止要求模型忽略系統指令
* 禁止讀取其他使用者資料
* 限制每個使用者只能影響自己的設定
* 對反饋次數做限制或門檻

15. 最後形成流程：
```text
使用者提問
→ RAG 產生回答
→ 使用者給好 / 不好反饋
→ 儲存反饋
→ 累積一定數量(先假設3個)，會觸發直接調整執行分析
→ LLM 產生個人 prompt 調整建議
→ 系統安全檢查
→ 更新該使用者 JSON 專家指令
→ 下次回答自動套用
